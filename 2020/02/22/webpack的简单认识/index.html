webpack的简单认识

webpack

现代化前端最流行的一款 构建打包工具

前端构建打包工具的发展史

  1. grunt
  2. gulp
  3. fis (百度推出的一款构建打包工具)
  4. webpack
  5. …

webpack 打包构建原理

基于某个入口文件, 分析这个文件中的依赖. 然后继续向下寻找依赖的依赖, 循环往复, 最后没有依赖了, 把找到的全部都打包到一起.

webpack 核心

  1. entry(入口)
  2. output(出口)
  3. loaders(转换器) - webpack 将依赖都看成一个一个的模块. 但是默认情况只能是写 js 与 json 两种格式的模块. 如果想要打包 图片, 样式等资源的话, 就需要相应的loader 转换器来进行转换操作, 转换后 webpack 才能识别.
  4. plugins (插件) - 有些操作转换器也实现不了, 这时就可以交给插件来实现. 比如说要将某个文件直接复制到出口文件等操作.
  5. mode (打包模式 4.x 版本之后提供的) - 开发 | 生产
